## 第 1 章：你的优势盘点（如何在面试讲故事）

把简历里的项目重新组织成 5 个**专家级故事**，每个故事都遵循 **STAR-T 模板**（Situation → Task → Action → Result → Tradeoff）：

### 故事 1：RISC-V Asample 推进（CR60Light）→ 体现"技术前瞻性 + 跨厂商推动"

**面试讲法**：
> "在博世 CR60Light 项目里我推动了基于芯来 RISC-V Asample 的方案验证。这件事的背景是 TI AWR2E44 的 R5F 核虽然成熟，但许可成本和长期生态我们判断有风险。我评估了三个候选——继续 ARM Cortex-R 路线、迁移到 RISC-V Asample、或者用 Cortex-M7 + 软件多核——最后选 RISC-V 是因为它能解决 IP 自主+成本+长期生态三个问题。
>
> 推进过程中最难的是说服测试和算法团队接受新工具链。我做了三件事：一是搭建了一套从 GCC 工具链到 OpenOCD 调试的最小可复现环境，二是写了一份 ABI 和中断行为差异的迁移指南，三是和芯来对接修了两个 PendSV 等价机制的 bug。
>
> 最终在 2 个月内完成了概念验证，确认调度延迟和 ARM 路线偏差 < 5%，这个评估报告作为内部决策依据。"

**关键词**（面试官会接的钩子）：RISC-V 中断模型 vs ARM、PendSV 在 RISC-V 上的等价物、CSR 寄存器、向量扩展、ABI 差异。

### 故事 2：SFR2K 平台框架从 0 到 1（NXP S32R45）→ 体现"架构设计 + 平台化思维"

**面试讲法**：
> "在 SFR2K 项目我从 0 到 1 搭建了普华 RTOS 之上的平台框架。当时痛点是每个项目都在重复搭建 BSP 和应用层粘合代码，每条产品线维护三套相似的代码导致 bug 修了又修。
>
> 我设计的方案是分四层：硬件抽象层（HAL）、RTE 层（屏蔽 RTOS 差异）、平台服务层（OEM 通用功能）、APP 层（OEM 定制）。用 Kconfig + Makefile 做编译期配置，目标是一份代码 base 支持 N 个 OEM 项目。
>
> 关键决策点有三个——第一是为什么不用 AUTOSAR CP，因为我们的 SoC 资源紧、AUTOSAR 工具链成本高；第二是为什么自研 RTE 而不直接基于 OS API，因为后续我们要支持 RTOS 切换；第三是为什么用 Kconfig 而不是 CMake option，因为 Kconfig 的依赖关系和默认值机制对硬件配置更友好。
>
> 这套框架后来支撑了三个量产项目，新项目 BSP 适配从 6 周降到 2 周。"

**关键词**：HAL/RTE 分层依据、Kconfig 内部机制、make 增量编译原理、平台化的边界在哪。

### 故事 3：A 核 DSP 下片调度（NXP R45）→ 体现"性能优化 + 硬件理解"

**面试讲法**（机制级深度）：
> "我们在 R45 上做的是把感知算法从 R 核下沉到 A 核 + DSP，目标是 15Hz 周期处理 4D 雷达点云。
>
> 关键瓶颈在 SRAM 容量和 DSP-SRAM 总线带宽。我做了三层优化：
>
> 第一层是缓存策略——把算法张量按 cache line（64B）对齐重排，减少 false sharing。这里有个细节，A53 的 L2 cache 是 PIPT，所以多核间 line 共享的 invalidate 开销不是按 VA 算的，必须从物理地址布局想。
>
> 第二层是流水线——把"DMA 输入 → 算子计算 → DMA 输出"做成三级流水，不同 stage 用不同 SRAM bank 错开总线竞争。这里关键是 DMA 描述符要预排链表，否则 DMA 完成中断 → CPU 配置下一笔的延迟会破坏流水节拍。
>
> 第三层是 SIMD——用 NEON intrinsics 把 conv2d 的 inner loop 写成 fp16x8 一拍 8 个元素。这里踩过一个坑，编译器自动向量化在 -O3 下对小尺寸循环反而做不好，需要 #pragma 强制循环展开。
>
> 最终时延从 80ms 压到 55ms，SRAM 峰值占用从 1.2MB 压到 768KB，满足 15Hz 余裕 30%。"

**关键词**：cache 一致性协议（MESI、MOESI）、PIPT vs VIPT、DMA scatter-gather、NEON 指令集、IPC 卡顿排查。

### 故事 4：UDS + 功能安全（Aurix TC357）→ 体现"汽车电子合规深度"

> 简历有现成的内容，但需要把"实现 UDS 0x22/0x2E/0x31"升级为"基于 ISO 14229，主导 UDS 状态机设计，对接整车 OEM 的 CDD 规范，处理过 Tester Present 与 Diagnostic Session 切换的并发竞态——这个 bug 是因为 Vector DaVinci 生成的 stub 在 0x10 切换瞬间没有正确锁住 0x27 的安全等级"。

**关键词**：ISO 14229 状态机、ISO 26262 ASIL 等级、Lockstep 双核、E2E 保护、SBC 看门狗。

### 故事 5：OpenAMP 双核通信（Zynq UltraScale+）→ 体现"系统级架构权衡"

> 这是你强项，DJI 准备已经覆盖。字节面试要补充的是：**为什么选 OpenAMP 不选其他**——和私有 IPC、Hypervisor、AMP+SMP 混合模式做对比，给出资源/实时性/可移植性维度的权衡矩阵。

## 第 2 章：必须补的短板（端侧 AI 推理）

这是岗位 2 的硬需求，**直接缺口**。30 天内你不可能成为专家，但要做到能跟面试官有内容地对话 30 分钟。

### 2.1 必须能讲清楚的 8 个概念

1. **TFLM（TensorFlow Lite Micro）架构**：interpreter / op resolver / memory arena / kernel registration 四件套
2. **Memory Arena**：为什么要静态预分配？persistent buffer vs scratch buffer 的区别？为什么 TFLM 不用 malloc？
3. **量化（Quantization）**：FP32 → INT8 的对称/非对称量化、scale 和 zero point 怎么算、为什么 conv 后接 ReLU 可以"折叠"
4. **算子裁剪**：op resolver 的作用、为什么不要 `AllOpsResolver`、按需注册算子能省多少 flash
5. **SIMD/NEON 优化**：fp16x8 / int8x16 的吞吐、`vmlaq_f32` 这类 intrinsics、对齐和尾部处理
6. **算子融合**：Conv + BN + ReLU 折叠、Depthwise Separable Conv 的实现技巧
7. **数据布局**：NCHW vs NHWC、为什么端侧通常用 NHWC（cache 友好）
8. **延迟 vs 吞吐 vs 内存的三角权衡**：什么时候用 batch=1、什么时候做 in-place、什么时候用 winograd

### 2.2 30 天补强路径

| 周次 | 任务 | 产出 |
|---|---|---|
| 第 1 周 | 读完 TFLM 官方架构文档 + 对比 你做过的 Halo-8 部署 | 写一份 1 页"TFLM 与 Halo-8 SDK 异同分析"的笔记 |
| 第 2 周 | 在 STM32F7 或 Cortex-M55 模拟器上跑一个 hello_world / micro_speech | 能在 GDB 里讲清 invoke() 的执行栈 |
| 第 3 周 | 看一遍 CMSIS-NN 的 SIMD conv 实现 | 能讲 INT8 卷积的 SIMD 内层循环 |
| 第 4 周 | 模拟面试题串讲 | 能 30 分钟从硬件不间断讲到算子优化 |

### 2.3 GPGPU / CUDA 兜底（兼投岗位 3 用）

不要求你成为 CUDA 工程师，但要会以下："SIMT 编程模型与 SIMD 的区别"、"warp / block / grid 的层次"、"共享内存 vs 全局内存的访存延迟"、"为什么 GPGPU 适合 dense 矩阵运算"、"NPU 相比 GPGPU 在端侧的优势（能耗比）"。能围绕这 5 点对答即可。

### 4.3 交叉面 / Leader 面（高级别 leader，45-60min）

这一轮**主要不是考技术**，是看：**判断力、价值观、能否带人、能否跨团队**。

**高频问题**：
- 描述一次你和别人意见不一致最后怎么解决的
- 描述一次你做错的技术决策和你怎么补救的
- 你怎么看 RTOS 在 AI 时代的角色变化
- 你为什么从汽车换到互联网公司
- 你下一个 3 年的技术规划

**核心打法**：
- 不要装完美。讲一个"我当时坚持 X 方案，后来证明 Y 方案是对的，我学到了……"的故事比"我从来没失败过"加分十倍。
- 体现对业务的理解。字节做端侧硬件不是为了硬件本身，是为了**支撑 AI 应用落地**。你的回答要体现这个视角。
- 不要在汽车 vs 互联网上抱怨前公司。讲你在追求什么——更前沿的技术栈、更快的迭代节奏、更大的算力规模。

### 4.4 HR 面（30min）

- 薪资谈判：字节专家级（3-1）的总包通常在 60-90W（取决于股票），不要主动报数，让 HR 先开。
- 离职原因：用"主动选择"的语气而不是"被动逃离"。
- 加班接受度：诚实但有底线。

## 第 5 章：跨团队 Owner 表达专项训练

这是你 DJI 模拟面试识别出的成长点，**对 L3 至关重要**。

### 5.1 错误表达 vs 正确表达对照表

| 场景 | ❌ 弱表达（L1-L2 风格） | ✅ 强表达（L3 风格） |

## 第 7 章：高优反问问题（让面试官记住你）

每轮面试结尾都会问"你有什么想问的"，**反问质量决定记忆度**。下面问题按场景分类：

**问技术深度（一面）**：
- 团队目前在 RTOS 上踩过最难的坑是什么？
- 端侧推理这块，你们自研框架还是基于 TFLM 二次开发？为什么这么选？
- 多核异构的内存模型你们是怎么解决一致性问题的？

**问架构与定位（二面）**：
- 这个团队在字节大盘里的定位是什么？1 年内要交付什么？
- 团队对"系统工程师"的能力模型是怎么定义的？哪些是入门门槛、哪些是 senior 的差异化？
- 团队和算法团队、硬件团队的边界怎么划？

**问文化与成长（leader 面）**：
- 团队目前最缺的能力是什么？
- 你看一个 senior 工程师 1 年内成长到 staff 的关键是什么？
- 团队对失败的态度是怎样的？有没有一个让团队学到很多的失败案例？

**不要问的问题**：
- 工资和福利（HR 面问）
- 加班情况（不要直接问，从工作节奏侧面问）
- 太基础的"团队多少人"
